# Статический IP для постоянного токена AppRoute — гайд для чайника

**Зачем это всё:**
AppRoute даёт два типа токенов — **временный** (работает без всего) и **постоянный**
(требует «белый список IPv4», то есть просит указать конкретный IP-адрес, с которого
будут идти запросы). Наш сайт живёт на Vercel, а Vercel **не даёт постоянного IP** —
каждый запрос уходит с нового адреса. AppRoute такие запросы будет отклонять.

**Решение простыми словами:**
Мы арендуем маленький сервер (VPS) — у него есть один постоянный IP. Запросы к AppRoute
пойдут не напрямую с Vercel, а «через» этот сервер. AppRoute видит всегда один и тот же
IP (адрес нашего сервера) — и пускает.

```
Было:   Vercel  →  AppRoute          ❌ IP всё время разный, постоянный токен не пускает
Стало:  Vercel  →  наш VPS  →  AppRoute   ✅ AppRoute видит один IP (наш VPS) — пускает
```

Код для этого **уже готов**. Тебе нужно только сделать сервер и вписать пару строк
настроек. Ниже — по шагам, медленно и подробно.

> ⚠️ **Это нужно ТОЛЬКО для постоянного токена.** Пока у тебя временный токен —
> ничего из этого делать не надо, всё и так работает. Вернись к гайду, когда AppRoute
> выдаст постоянный токен.

---

## Что понадобится

- Банковская карта (для аренды сервера, ~150–300 ₽/мес).
- Доступ к панели Vercel (там, где задеплоен сайт).
- Постоянный токен AppRoute и его Base URL (выдаёт заказчик/AppRoute).
- 30–40 минут времени.

---

## Шаг 1. Арендовать сервер (VPS)

VPS — это «удалённый компьютер», который всегда включён и имеет постоянный IP-адрес.

1. Зайди на сайт хостинга. Подойдёт любой, для примера — **aeza.net** (есть оплата
   рос. картами) или **timeweb.cloud**.
2. Зарегистрируйся, привяжи карту.
3. Создай новый VPS с такими параметрами (бери **самый дешёвый**, мощности не нужны):
   - **Операционная система:** `Ubuntu 24.04` (или 22.04 — без разницы).
   - **Тариф:** минимальный (1 ядро, 1 ГБ RAM — за глаза).
   - **IP:** убедись, что в тарифе есть **выделенный (статический) IPv4**. Обычно он
     уже включён. Это самое главное — без него смысла нет.
4. После создания хостинг покажет:
   - **IP-адрес** сервера — например `203.0.113.45`. **Запиши его.**
   - **Пароль root** (или предложит создать) — **запиши.**

> 💡 Запиши IP и пароль в надёжное место — они понадобятся ещё несколько раз.

---

## Шаг 2. Подключиться к серверу

Нужно открыть «терминал» сервера. Это окно, куда вводят команды.

### Если у тебя Windows
1. Нажми `Пуск`, набери `PowerShell`, открой его.
2. Введи команду (подставь свой IP вместо `203.0.113.45`):
   ```
   ssh root@203.0.113.45
   ```
3. Спросит `Are you sure you want to continue connecting?` — набери `yes`, Enter.
4. Спросит пароль — вставь пароль root (при вводе пароль **не отображается**, это
   нормально, просто вставь и нажми Enter).
5. Если увидел строку вида `root@...:~#` — ты внутри сервера. 🎉

### Если у тебя Mac
То же самое, но терминал называется `Terminal` (Программы → Утилиты → Терминал).
Команда `ssh root@203.0.113.45` такая же.

---

## Шаг 3. Установить «прокси» (tinyproxy)

Прокси — это программа-посредник, которая принимает запрос и пересылает дальше.
Именно она будет «прокладкой» между Vercel и AppRoute.

Внутри сервера (там, где `root@...:~#`) введи по очереди, **каждую строку отдельно**,
после каждой — Enter:

```bash
apt update
```
```bash
apt install -y tinyproxy
```

Подожди, пока установится (несколько секунд).

---

## Шаг 4. Настроить прокси (логин и пароль)

Прокси нужно закрыть паролем, иначе им сможет пользоваться кто угодно.

1. Открой файл настроек командой:
   ```bash
   nano /etc/tinyproxy/tinyproxy.conf
   ```
   Откроется простой текстовый редактор прямо в терминале.

2. Файл длинный. Тебе нужно найти и поправить несколько строк. В `nano` искать можно
   через `Ctrl+W` (ввести слово → Enter).

3. **Найди строку `Port 8888`** — убедись, что она есть и не начинается с `#`
   (решётка = строка отключена). Обычно она уже правильная. Должно быть просто:
   ```
   Port 8888
   ```

4. **Найди строки, начинающиеся с `Allow`** (например `Allow 127.0.0.1`).
   Перед каждой такой строкой поставь решётку `#`, чтобы отключить ограничение по IP
   (мы будем защищаться паролем, а не IP, потому что у Vercel адрес меняется):
   ```
   #Allow 127.0.0.1
   #Allow ::1
   ```

5. **В самый конец файла** (пролистай стрелкой вниз до конца) добавь две строки.
   Придумай свой логин и **длинный пароль** (буквы+цифры, без пробелов):
   ```
   Listen 0.0.0.0
   BasicAuth approuteuser MoiOchenDlinnyParol2026xYz
   ```
   - `approuteuser` — логин (можешь оставить такой).
   - `MoiOchenDlinnyParol2026xYz` — **замени на свой пароль**, запиши его.
   - `Listen 0.0.0.0` означает «принимать запросы извне».

6. Сохрани и выйди: нажми `Ctrl+O` → Enter (сохранить), потом `Ctrl+X` (выйти).

7. Перезапусти прокси, чтобы настройки применились:
   ```bash
   systemctl restart tinyproxy
   ```
   ```bash
   systemctl enable tinyproxy
   ```
   (вторая команда — чтобы прокси сам включался после перезагрузки сервера).

---

## Шаг 5. Открыть порт в файрволе

Файрвол — «охранник» сервера, который по умолчанию может блокировать соединения.
Разрешим наш порт `8888`:

```bash
ufw allow 8888/tcp
```
```bash
ufw allow OpenSSH
```
```bash
ufw --force enable
```

> Вторая команда (`OpenSSH`) обязательна — иначе ты потеряешь доступ к серверу по SSH.

---

## Шаг 6. Проверить, что прокси работает

Не выходя из сервера, проверь сам себя (подставь свои логин:пароль):

```bash
curl -x http://approuteuser:MoiOchenDlinnyParol2026xYz@127.0.0.1:8888 https://api.ipify.org
```

Если в ответ вывелся **IP-адрес твоего сервера** (тот самый, например `203.0.113.45`) —
**всё работает!** 🎉 Прокси на месте и отдаёт правильный IP.

Если ошибка — вернись к Шагу 4, проверь, что нет лишних решёток на `Port`, `Listen`,
`BasicAuth`, и что ты перезапустил `systemctl restart tinyproxy`.

Теперь можно выйти из сервера: набери `exit` и Enter.

---

## Шаг 7. Вписать IP сервера в белый список AppRoute

1. Зайди в личный кабинет AppRoute (туда, где создаются токены).
2. Создавай/настраивай **постоянный** токен.
3. В поле «белый список IPv4» впиши **IP твоего сервера** — точным значением,
   например:
   ```
   203.0.113.45
   ```
   (не wildcard, не диапазон — именно один точный адрес твоего VPS).
4. Сохрани. Скопируй сам **постоянный токен** и **Base URL** — понадобятся дальше.

---

## Шаг 8. Прописать настройки в проекте

Теперь нужно сказать нашему коду: «ходи в AppRoute через этот прокси, с этим токеном».
Это делается в двух местах — **локально** и **на Vercel**.

### Строка прокси

Сначала собери строку прокси (понадобится дважды). Формат:
```
http://ЛОГИН:ПАРОЛЬ@IP_СЕРВЕРА:8888
```
Например:
```
http://approuteuser:MoiOchenDlinnyParol2026xYz@203.0.113.45:8888
```

### А) Локально (файл `.env.local` в проекте)

Открой файл `.env.local` в корне проекта и найди блок AppRoute. Заполни три строки:
```
APPROUTE_API_KEY=<сюда постоянный токен>
APPROUTE_BASE_URL=<сюда Base URL от AppRoute>
APPROUTE_OUTBOUND_PROXY=http://approuteuser:MoiOchenDlinnyParol2026xYz@203.0.113.45:8888
```

### Б) На Vercel (чтобы работало на «боевом» сайте)

1. Зайди на **vercel.com** → твой проект (`nice-try-blush`).
2. Вкладка **Settings** → слева **Environment Variables**.
3. Добавь/обнови **три** переменные (кнопка `Add New`), для каждой выбери окружение
   **Production** (и Preview, если нужно):

   | Name (имя)                | Value (значение)                                              |
   |---------------------------|--------------------------------------------------------------|
   | `APPROUTE_API_KEY`        | постоянный токен AppRoute                                     |
   | `APPROUTE_BASE_URL`       | Base URL от AppRoute                                          |
   | `APPROUTE_OUTBOUND_PROXY` | `http://approuteuser:MoiOchenDlinnyParol2026xYz@203.0.113.45:8888` |

4. Нажми **Save** для каждой.
5. **Важно:** Vercel применяет новые переменные только при следующем деплое.
   Зайди во вкладку **Deployments** → у последнего деплоя меню (три точки) →
   **Redeploy** → подтверди. Подожди пару минут.

---

## Шаг 9. Проверить, что всё заработало (боевой режим)

В терминале **в папке проекта** (на своём компьютере, не на сервере) запусти:

```bash
npm run sync:approute
```

- Если вверху написано **«AppRoute: БОЕВОЙ режим (реальный API)»** и каталог реально
  загрузился (без ошибок) — **готово, всё работает через прокси!** ✅
- Если написано «ФОЛБЭК (мок-каталог)» — значит одна из трёх переменных пустая или
  с опечаткой. Проверь `.env.local`.
- Если ошибка сети/таймаут — проверь прокси (Шаг 6) и что IP вписан в вайтлист (Шаг 7).

## Частые проблемы

**«Ничего не работает, ФОЛБЭК».**
Значит код считает, что данных нет. Проверь, что в `.env.local` заполнены и
`APPROUTE_API_KEY`, и `APPROUTE_BASE_URL` (без `your_...`-заглушек), и что Base URL
начинается с `https://`.

**«Ошибка авторизации / 401 от AppRoute».**
Либо неверный токен, либо IP сервера не совпадает с тем, что в вайтлисте. Проверь Шаг 7:
впиши ровно тот IP, что показал `curl ... api.ipify.org` на Шаге 6.

**«Потерял доступ к серверу по SSH».**
Скорее всего файрвол закрыл доступ. Через панель хостинга есть «веб-консоль»
(VNC/Console) — зайди через неё и выполни `ufw allow OpenSSH`.

**«Запросы медленные».**
Бери VPS поближе (Европа/Россия). Для пары запросов в AppRoute задержка некритична.

**Безопасность.**
Прокси защищён логином и паролем — никому их не давай и не коммить в git. Они живут
только в `.env.local` (он в `.gitignore`) и в настройках Vercel.

---

## Когда выдадут… ничего больше делать не надо

Если AppRoute поменяет токен — просто обнови `APPROUTE_API_KEY` в `.env.local` и на
Vercel (+ Redeploy). Сервер и прокси трогать не нужно, они работают как есть.

Если решишь вернуться на временный токен (без вайтлиста) — оставь
`APPROUTE_OUTBOUND_PROXY` пустым, и запросы снова пойдут напрямую. Код это понимает
автоматически.
