# Чек-лист быстрой ручной проверки — NiceTry

> Прогон ~10–15 мин. Запуск: `npm run dev` (или `start.bat`) → http://localhost:3000

## 0. Сборка
- [ ] `npm run build` проходит без ошибок
- [ ] `npm run type-check` — без ошибок TypeScript
- [ ] Главная страница открывается (виден Header + Footer + контент, не пусто)

## 1. Авторизация (magic link)
- [ ] `/auth/login` — форма принимает email, шлёт ссылку
- [ ] Переход по magic link логинит, редиректит назад
- [ ] В Header после входа видны баланс и статус
- [ ] `/profile` — email, статус, баланс, реферальный код (кнопка «копировать» работает)
- [ ] Профиль создаётся автоматически при первом входе (запись в `users`, код 8 символов, статус Bronze)
- [ ] Выход (logout) работает, сессия сбрасывается

## 2. Защита роутов (middleware)
- [ ] Аноним на `/profile`, `/orders`, `/balance` → редирект на `/auth/login`
- [ ] Аноним на `/admin/*` → редирект
- [ ] Не-админ на `/admin/*` → редирект (проверка `is_admin`)

## 3. Каталог
- [ ] Витрина показывает товары (10 категорий, ~34 товара) даже без ключей API (фолбэк-каталог)
- [ ] Фильтры и поиск работают
- [ ] Карточки 4 типов выдачи отображаются корректно
- [ ] Страница товара `/product/[id]` — описание, цена, доп. поля заполняются
- [ ] Цена соответствует ТЗ: Steam $10 = **912 ₽**, $5 = **456 ₽**

## 4. Корзина и заказ
- [ ] Добавление/удаление товара в корзине, счётчик в Header
- [ ] Применение промокода (валидация в `/api/promo/validate`)
- [ ] Скидка по статусу пользователя учитывается
- [ ] Суммы пересчитываются на сервере (клиенту не доверяем)
- [ ] Оплата с внутреннего баланса (Контур B) — заказ создаётся, баланс списывается
- [ ] Оплата картой/крипто → возвращает **501** (пауза перед платежами — это ожидаемо)

## 5. Админ-панель (`/admin`)
- [ ] Дашборд открывается, цифры не пустые
- [ ] Товары: список / создание / редактирование / ключи
- [ ] Заказы: список, детали, возврат (refund)
- [ ] Пользователи: список, правка баланса (нужна причина, транзакция пишется со знаком)
- [ ] Промокоды: список / создание / правка
- [ ] Статусы: список / создание / правка
- [ ] Все списки заполнены (admin-API работает через service-role, не пустые под RLS)

## 6. Безопасность (разовая проверка)
- [ ] `.env*.local` не в Git; service-role ключ без `NEXT_PUBLIC`
- [ ] ⚠️ Применён `supabase_security.sql` в Supabase SQL Editor (иначе витрина читает только фолбэк, не БД)
- [ ] `/api/orders/[id]` отдаёт заказ только владельцу или админу
